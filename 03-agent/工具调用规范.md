## 2. RAG Search Tool

### 2.1 Interface Definition

```yaml
tool: rag_search
description: Search relevant content in the knowledge base
parameters:
  query:
    type: string
    description: Search query text
    required: true
  knowledge_base:
    type: enum
    enum: ["laws", "cases", "contract_templates", "review_rules"]
    description: Target knowledge base
    required: true
  strategy:
    type: enum
    enum: ["semantic", "keyword", "hybrid"]
    default: "hybrid"
    description: Search strategy
  top_k:
    type: integer
    default: 5
    minimum: 1
    maximum: 20
    description: Number of results to return
  filters:
    type: object
    description: Multi-dimensional filter conditions
    properties:
      law_type:
        type: string
        description: Legal source filter (e.g., "Civil Code")
      court_level:
        type: array
        description: Court level filter
      year_range:
        type: array
        items: integer
        description: Year range filter
  output_format:
    type: enum
    enum: ["full", "summary"]
    default: "full"
    description: Return detailed or summary format
```

### 2.2 Usage Example

```python
# Python SDK call
result = rag_search(
    query="Liquidated damages must not exceed 30% of losses",
    knowledge_base="laws",
    strategy="hybrid",
    top_k=5,
    filters={"law_type": "Civil Code"},
    output_format="full"
)
```

## 3. LLM Calling Tool

### 3.1 Standard Call

```python
# Standard LLM call (through unified wrapper layer)
response = llm_chat(
    model="mimo2.5",  # or "deepseek-v4-flash"
    system_prompt=AGENT_SYSTEM_PROMPTS["analyzer"],
    user_message=build_analyzer_message(clause_json),
    temperature=0.1,  # Legal analysis requires low temperature (high determinism)
    max_tokens=2000,
    response_format={"type": "json_object"}  # Force JSON output
)
```

### 3.2 Model Selection Strategy

| Task | Model | Rationale |
|---|---|---|
| Document parsing (non-table) | DeepSeek V4-Flash | Minimal cost; sufficient for text processing |
| Document parsing (complex tables) | MiMo 2.5 | Strong table comprehension |
| Legal clause analysis | MiMo 2.5 | High reasoning requirement |
| Report generation | DeepSeek V4-Flash | Structured text generation, low cost |
| Report validation | MiMo 2.5 | Quality checks cannot afford errors |

### 3.3 Token Consumption Estimate


## 4. Fault Tolerance & Degradation

```python
# LLM call with built-in retry and degradation
def llm_chat_with_fallback(**kwargs):
    primary_model = kwargs.get("model", "gpt-4o")
    fallback_models = ["deepseek-v3", "claude-3.7-sonnet"]
    
    models_to_try = [primary_model] + fallback_models
    
    for model in models_to_try:
        try:
            kwargs["model"] = model
            result = _call_llm_api(**kwargs)
            return result
        except RateLimitError:
            time.sleep(2)
            continue
        except ServiceUnavailableError:
            continue
        except TimeoutError:
            continue
    
    raise AllLLMsUnavailable("All LLM models are unavailable")
```
