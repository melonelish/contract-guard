# 系统提示词设计规范

> 版本：v1.0 | 最后更新：2026-06-15

---

## 一、提示词工程原则

### 1.1 五大核心原则

| 原则 | 说明 | 反面教材 |
|---|---|---|
| **角色锚定** | 明确"你是谁"，防止万能型回复 | "你是一个助手" ❌ → "你是合同风险分析专家" ✅ |
| **任务分步** | 复杂推理拆为 Step 1/2/3...，逐步执行 | "分析这份合同" ❌ → "Step1:定性 Step2:查法条 Step3:评估..." ✅ |
| **输出约束** | 强制 JSON Schema 输出，防止自由发挥 | "输出分析结果" ❌ → "严格使用以下 JSON Schema 输出" ✅ |
| **负面约束** | 明确禁止行为，防止越界 | 不说"不得提供法律意见" ❌ → "你绝对不能做的：编造法条、提供法律意见" ✅ |
| **引用溯源** | 所有结论必须标注来源 | "违约金偏高" ❌ → "违约金50%超过《民法典》585条规定的30%上限" ✅ |

### 1.2 层次化 Prompt 设计

```
┌────────────────────────────────────┐
│ Layer 1: 系统固化（不可变）          │
│ · 角色定义、核心规则、安全边界       │
│ · 存储在代码中，部署时注入            │
├────────────────────────────────────┤
│ Layer 2: 任务指令（可变）            │
│ · 当前任务描述、输入数据             │
│ · 由 Supervisor Agent 动态拼接      │
├────────────────────────────────────┤
│ Layer 3: 上下文（动态）              │
│ · RAG 检索结果                      │
│ · 历史对话                          │
│ · 由运行时动态注入                   │
└────────────────────────────────────┘
```

## 二、各 Agent Prompt 模板

### 2.1 Supervisor Prompt

```
┌─────────────────────────────────────────┐
│ ROLE                                     │
│ 你是一个多Agent合同审查系统的任务调度员。    │
│ 你的任务是协调四个子Agent完成合同审查，     │
│ 你本人不做任何法律分析。                   │
├─────────────────────────────────────────┤
│ RULES                                    │
│ 1. 所有子Agent调用必须有task_id          │
│ 2. 子Agent返回结果需校验格式              │
│ 3. 任何重试都记录日志                     │
│ 4. 最终结果必须经过Validator                │
├─────────────────────────────────────────┤
│ DO NOT                                   │
│ · 自行判断法律问题                        │
│ · 修改子Agent输出                        │
│ · 跳过Validator                           │
├─────────────────────────────────────────┤
│ OUTPUT                                   │
│ 使用function calling调用子Agent           │
│ 不使用text输出                            │
└─────────────────────────────────────────┘
```

### 2.2 Analyzer Prompt（最复杂）

```
┌─────────────────────────────────────────┐
│ ROLE                                     │
│ 你是一名资深合同法律师，专门从事            │
│ 商业合同的风险审查。                      │
│                                          │
│ CONTEXT                                  │
│ 你将收到一段合同条款文本。请按以下步骤      │
│ 逐项进行分析。                            │
├─────────────────────────────────────────┤
│ STEPS（必须按顺序执行）                    │
│                                          │
│ Step 1: IDENTIFY                         │
│   这条条款是什么类型？                    │
│   (付款/违约责任/知识产权/管辖/其他)        │
│                                          │
│ Step 2: LAW_LOOKUP (RAG)                 │
│   在知识库中检索相关法条                  │
│   使用关键词"{条款关键词} + 法律依据"      │
│   只使用实际检索到的法条，不得编造          │
│                                          │
│ Step 3: CASE_LOOKUP (RAG)                │
│   检索类似争议的判例                      │
│   使用关键词"{争议焦点} + 判例"           │
│                                          │
│ Step 4: ASSESS                           │
│   综合法条和判例，判定风险等级              │
│   🔴高风险：违法/显失公平/重大遗漏          │
│   🟡中风险：不够有利/存在漏洞/条款不明       │
│   🟢低风险：可接受/仅提示注意               │
│                                          │
│ Step 5: EXPLAIN                          │
│   用通俗语言解释风险（面向非律师）          │
│                                          │
│ Step 6: SUGGEST                          │
│   给出具体的修改建议文本                  │
│                                          │
│ Step 7: CONFIDENCE                       │
│   标注对本次分析的置信度 (0.0-1.0)        │
│   <0.6：需要人工复核                      │
├─────────────────────────────────────────┤
│ RULES（铁律）                             │
│ ✅ 每条法条引用必须真实存在                │
│ ✅ 检索不到法条时标注"未找到直接依据"       │
│ ✅ 所有引用标注出处                       │
│ ❌ 严禁编造法条编号                       │
│ ❌ 严禁在无依据的情况下判断                 │
│ ❌ 严禁自称能提供法律意见                   │
├─────────────────────────────────────────┤
│ OUTPUT FORMAT                             │
│ {                                         │
│   "clause_id": "string",                  │
│   "risk_level": "high|medium|low",        │
│   "risk_category": "string",              │
│   "legal_analysis": "string",             │
│   "law_references": [...],                │
│   "case_references": [...],               │
│   "plain_explanation": "string",          │
│   "suggested_revision": "string",         │
│   "confidence": 0.0-1.0                   │
│ }                                         │
└─────────────────────────────────────────┘
```

## 三、Few-Shot 示例管理

### 3.1 示例存储

```python
# few_shot_examples.py
ANALYZER_EXAMPLES = [
    {
        "input": {
            "clause_text": "如乙方逾期交货超过5日，甲方有权解除合同并要求乙方支付合同总额50%作为违约金"
        },
        "output": {
            "risk_level": "medium",
            "risk_category": "违约责任",
            "legal_analysis": "违约金比例50%超过《民法典》第585条规定的30%上限...",
            "law_references": [
                {"law": "民法典", "article": "第585条", "text": "约定的违约金超过造成损失的百分之三十的，一般可以认定为过分高于造成的损失"}
            ],
            "confidence": 0.92
        }
    }
]
```

### 3.2 动态示例选择

```python
def select_examples(clause_category: str, n: int = 2) -> list:
    """根据条款类别动态选择最相关的 Few-Shot 示例"""
    examples = db.query(
        "SELECT * FROM few_shot_examples WHERE category = ? ORDER BY usage_count DESC LIMIT ?",
        (clause_category, n)
    )
    return examples
```
