# 实现规格说明书模板

> **用途**：本文档是给 AI 编码 Agent（如 Claude）直接执行的实现规格。
> Agent 读完本文档 + 现有架构文档后，应能直接写出可运行的代码，无需猜测。
>
> **填写原则**：只写"做什么"和"怎么接"，不写"为什么"（"为什么"在架构文档里）。

## 四、状态机定义

### 4.1 审查状态机

```python
# backend/app/core/state_machine.py

from enum import Enum

class ReviewStatus(str, Enum):
    CREATED = "created"
    QUEUED = "queued"
    PARSING = "parsing"
    PARSED = "parsed"
    ANALYZING = "analyzing"
    ANALYZED = "analyzed"
    PARTIAL_FAIL = "partial_fail"
    REPORTING = "reporting"
    REPORTED = "reported"
    VALIDATING = "validating"
    APPROVED = "approved"
    REJECTED = "rejected"
    RETRYING = "retrying"
    COMPLETED = "completed"
    CANCELLED = "cancelled"
    FAILED = "failed"

# (当前状态, 事件) -> 下一个状态
TRANSITIONS = {
    # 正常流程
    ("CREATED",    "submit"):           "QUEUED",

### 8.2 FastAPI → Agent 调用链

```
HTTP Request
  → API Route Handler (v1/reviews.py)
    → ReviewOrchestrator.start()
      → 写入 DB (reviews 表)
      → redis.xadd("agent:parser:tasks", payload)
      → 返回 task_id 给前端

Worker 消费
  → AgentWorker.run() (workers/agent_worker.py)
    → 监听对应 Stream
    → 反序列化 AgentMessage
    → 实例化对应 Agent
    → agent.execute(task_context)
    → 结果写入 DB + redis.xadd 到下一个 Stream

Supervisor 监控
  → SupervisorAgent.check_completion()
    → 检查该 task_id 下所有 Agent 状态
    → 全部完成 → 触发下一阶段
    → 部分失败 → 决定重试/放弃
```

### 8.3 嵌入向量生成时机

```
1. Parser Agent 解析完条款后 → 调用 embed() 生成条款嵌入 → 存入 clauses.embedding
2. 知识库导入时 → 调用 embed() 生成法条嵌入 → 存入 law_articles.embedding
3. 审查触发时 → Analyzer Agent 先用条款嵌入做 RAG 检索 → 再分析
```

### 8.4 Drafter → Review 交接

```
1. Drafter 输出 → 存入 contract_drafts.draft_json
2. 用户在编辑器中修改 → PUT /contracts/{id}/draft 更新 draft_json
3. 用户点击"重新审查" → POST /contracts/{id}/draft/review
4. ReviewOrchestrator 收到请求 → 从 draft_json 重新提取条款 → 进入审查流程
5. 增量审查：对比 draft_json 与上一版本 → 只审查变更的条款（clause_changes 表）
```

class Settings(BaseSettings):
    # === 应用 ===
    APP_NAME: str = "ContractGuard"
    DEBUG: bool = False
    SECRET_KEY: str                            # 必填，64 字符
    CORS_ORIGINS: str = "http://localhost:5173"

    # === 数据库 ===
    DATABASE_URL: str                          # postgresql+asyncpg://...
    DATABASE_MAX_CONNECTIONS: int = 20

    # === Redis ===
    REDIS_URL: str = "redis://localhost:6379/0"

    # === LLM 模型 ===
    MIMO_API_KEY: str                          # 必填
    MIMO_BASE_URL: str = "https://api.mimo.com/v1"
    DEEPSEEK_API_KEY: str                      # 必填
    DEEPSEEK_BASE_URL: str = "https://api.deepseek.com/v1"

    # === Milvus ===
    MILVUS_HOST: str = "localhost"
    MILVUS_PORT: int = 19530

    # === Elasticsearch ===
    ES_HOST: str = "http://localhost:9200"

    # === MinIO ===
    MINIO_ENDPOINT: str = "localhost:9000"
    MINIO_ACCESS_KEY: str = "minioadmin"
    MINIO_SECRET_KEY: str = "minioadmin"
    MINIO_BUCKET: str = "contractguard"

    # === Vault ===
    VAULT_ADDR: str = "http://localhost:8200"
    VAULT_TOKEN: str = ""                      # 开发环境用 token，生产用 AppRole

    # === JWT ===
    JWT_ALGORITHM: str = "HS256"               # 开发用 HS256，生产切 RS256
    JWT_ACCESS_TOKEN_EXPIRE_MINUTES: int = 1440
    JWT_REFRESH_TOKEN_EXPIRE_DAYS: int = 7

    # === Agent 配置 ===
    AGENT_MAX_CONCURRENCY: int = 10            # 并行分析最大并发
    AGENT_TOTAL_TPM: int = 100000              # 全局 token/分钟上限
    AGENT_PARSE_TIMEOUT: int = 120             # 秒
    AGENT_ANALYZE_TIMEOUT: int = 60
    AGENT_REPORT_TIMEOUT: int = 120
    AGENT_VALIDATE_TIMEOUT: int = 30

    # === 文件上传 ===
    MAX_UPLOAD_SIZE_MB: int = 50
    ALLOWED_MIME_TYPES: str = "application/pdf"

    # === 监控 ===
    PROMETHEUS_PORT: int = 9090

    model_config = {"env_file": ".env", "env_file_encoding": "utf-8"}
```

"""
Agent Worker 启动入口。

用法：
    python -m worker.run --agent parser --concurrency 2
    python -m worker.run --agent analyzer --concurrency 5
    python -m worker.run --agent report --concurrency 1
    python -m worker.run --agent validator --concurrency 1

每个 Worker 进程只监听一个 agent 的 Stream。
生产环境用 docker compose scale 启动多个 Worker 实例。
"""

# 启动流程：
# 1. 加载 Settings
# 2. 初始化 DB session 工厂
# 3. 初始化 Redis 连接池
# 4. 初始化 LLM Wrapper
# 5. 创建 Consumer Group（如果不存在）
# 6. 进入主循环：
#    a. redis.xreadgroup(group, consumer, streams, count=1, block=5000)
#    b. 反序列化 AgentMessage
#    c. 更新任务状态为 RUNNING
#    d. agent.execute(task_context)
#    e. 更新任务状态为 COMPLETED / FAILED
#    f. redis.xack(stream, group, message_id)
