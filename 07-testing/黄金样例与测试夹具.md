# 黄金样例与测试夹具

> 版本：v1.0 | 最后更新：2026-06-16

---

## 一、文档目的

本文件不是产品说明，而是给 AI 编码 Agent 的“对照答案”。

目标：

- 给出最小但完整的输入输出样例
- 降低 Agent 在 Parser / Analyzer / API / WebSocket 上的脑补空间
- 让后续实现可以直接按样例写单元测试、集成测试、回归测试

---

## 二、统一 ID 规则

| 对象 | 类型 | 规则 |
|---|---|---|
| `tenant_id` | UUID | 租户主键 |
| `user_id` | UUID | 用户主键 |
| `contract_id` | UUID | 合同主键 |
| `review_id` | UUID | 一次审查记录主键 |
| `task_id` | UUID | 一次异步编排任务主键；MVP 阶段与 `review_id` 一一对应 |
| `clause_id` | UUID | 条款主键 |
| `clause_code` | string | 面向前端展示和引用的稳定业务编号，如 `cl_007` |
| `version_id` | UUID | 合同版本主键 |
| `clause_analysis_id` | UUID | 条款分析记录主键 |

### 强制规则

- 数据库主键统一使用 UUID
- 面向 UI、报告、前端定位展示时，条款同时返回 `clause_id` 和 `clause_code`
- MVP 阶段 `review_id` 与 `task_id` 必须一一对应，不允许一个 `review_id` 绑定多个并行 `task_id`

## 四、WebSocket 黄金事件序列

### 4.1 订阅端点

```text
ws://host/ws/review/{review_id}
```

### 4.2 事件样例

```json
{"event":"stage","review_id":"55555555-5555-5555-5555-555555555555","task_id":"55555555-5555-5555-5555-555555555555","stage":"parsing","progress":10,"detail":"识别到18条条款"}
{"event":"stage","review_id":"55555555-5555-5555-5555-555555555555","task_id":"55555555-5555-5555-5555-555555555555","stage":"analyzing","progress":45,"detail":"正在分析条款 8/18","clause_current":8,"clause_total":18}
{"event":"stage","review_id":"55555555-5555-5555-5555-555555555555","task_id":"55555555-5555-5555-5555-555555555555","stage":"reporting","progress":90,"detail":"正在生成审查报告"}
{"event":"complete","review_id":"55555555-5555-5555-5555-555555555555","task_id":"55555555-5555-5555-5555-555555555555","summary":{"total_risks":4,"high":1,"medium":2,"low":1},"duration_sec":286}
```

### 4.3 强制规则

- WebSocket 路径统一使用 `review_id`
- 事件体必须同时带 `review_id` 和 `task_id`
- `stage` 枚举固定为 `parsing | analyzing | reporting | validating`
- `event` 枚举固定为 `stage | complete | failed`

## 五、反馈接口黄金样例

### 5.1 请求

```json
{
  "clause_analysis_id": "44444444-4444-4444-4444-444444444444",
  "vote": "accurate",
  "comment": "分析准确，建议可直接采用"
}
```

### 5.2 规则

- 反馈对象是 `clause_analysis_id`，不是 `clause_id`
- `vote` 枚举固定为 `accurate | inaccurate`
- MVP 不使用 `is_helpful`
- 若后续需要整份报告反馈，新增 `report_vote` 字段，不复用条款反馈结构

## 六、最小自动化测试集

| 测试 ID | 类型 | 目标 |
|---|---|---|
| T-API-001 | API | 上传合同成功返回 `contract_id` |
| T-API-002 | API | 触发审查成功返回 `review_id` 和 `task_id` |
| T-WS-001 | WebSocket | 审查过程按阶段推送进度 |
| T-PARSER-001 | 单测 | Parser 输出 `clause_id + clause_code + page + position` |
| T-ANALYZER-001 | 单测 | Analyzer 输出 `clause_analysis_id + risk_level + confidence` |
| T-FEEDBACK-001 | API | 反馈接口接收 `clause_analysis_id + vote` |
| T-RLS-001 | 安全 | A 租户不能读取 B 租户合同 |
