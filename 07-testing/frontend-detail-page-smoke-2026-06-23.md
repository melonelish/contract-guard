## 4. 发现的问题

### 问题 1：全局鉴权失败会先于详情页错误态触发

表现：
- 后端离线并整页刷新详情页时，页面被带回登录页
- 无法看到 `ContractDetailPage` 中新增的错误页

影响：
- 用户会感知为“应用要求重新登录”，而不是“合同详情加载失败”

判断：
- 这不是本轮详情页修复引入的新 bug
- 这是更上游的 `auth hydrate / auth me` 失败处理策略决定的行为

建议：
- 如果产品希望“后端离线时仍留在当前页面并显示局部错误”，需要在更上层做全局离线态设计，而不是继续在 `ContractDetailPage` 内修

### 问题 2：开发环境存在无关 HMR 报错

表现：
- 浏览器控制台有 `HomePreviewV2.tsx` 的 HMR reload 失败日志

影响：
- 不影响本轮详情页验收主流程
- 但会污染开发态日志，降低定位效率

建议：
- 后续单独清理该预览页的语法或模块引用问题

## 6. 残留风险

- 当前离线体验分成两层：
  - 全局鉴权层
  - 详情页数据加载层
- 详情页本身已经有较好的错误态，但全局鉴权失败会更早发生
- 因此，若后续要把“后端离线用户体验”真正收口，需要补一个更上层的全局离线/服务不可用状态设计
