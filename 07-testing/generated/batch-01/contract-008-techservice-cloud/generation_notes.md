# 合同8测试资产生成说明

## 合同背景

本合同模拟某大型传统制造企业将核心业务系统迁移至公有云的场景。项目总价520万元，项目周期8个月，属于大型云迁移项目。

**业务场景特点：**
- 甲方为上市公司，核心业务系统包括ERP、MES、CRM等，业务连续性要求高
- 云迁移涉及系统架构重大变更，技术风险高
- 迁移时间窗口有限（周末），业务中断风险大
- 数据迁移涉及大量核心业务数据，数据安全风险突出

## 矛盾设计意图

### C008-01：迁移时间窗口与项目计划矛盾
- **设计方式**：第3.3条要求周末迁移，第3.2条将测试与切换安排在同一阶段
- **测试目标**：验证AI能否识别时间窗口不足导致的业务中断风险
- **难度**：较高，需要理解云迁移的技术复杂性

### C008-02：责任免除与赔偿上限矛盾
- **设计方式**：第11.2条扩大免责范围，第11.6条限制赔偿上限
- **测试目标**：验证AI能否识别两个条款共同作用导致甲方保护不足的问题
- **难度**：中等

### C008-03：云资源费用承担与控制矛盾
- **设计方式**：第7.1条由甲方承担费用，第7.3条要求乙方合理使用但无标准
- **测试目标**：验证AI能否识别费用承担与控制机制不匹配的问题
- **难度**：中等

### C008-04：验收标准与测试基础矛盾
- **设计方式**：第5.2条要求性能达标，但缺乏性能基准测试条款
- **测试目标**：验证AI能否识别验收标准缺乏实施基础的问题
- **难度**：中等

## 测试价值说明

本合同测试资产具有以下测试价值：

1. **技术风险识别**：测试AI能否理解云迁移的技术复杂性，识别回滚方案、性能测试等关键条款的缺失
2. **责任条款分析**：测试AI能否识别免责范围过宽和赔偿上限过低的组合风险
3. **时间安排合理性评估**：测试AI能否识别迁移时间窗口与业务连续性的矛盾
4. **费用控制机制评估**：测试AI能否识别云资源费用缺乏控制机制的问题
5. **付款公平性判断**：测试AI能否识别付款条件严重偏向服务商的问题
