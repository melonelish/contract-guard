# 测试资产生成说明

## 一、合同背景

本合同为金融机构（股份制银行）IT设备采购合同，背景为核心业务系统信创改造项目。甲方采购国产化服务器、分布式存储及网络设备，替换现有进口设备集群，涉及核心交易系统、信贷管理系统及数据仓库三大业务系统的硬件底座升级。

**关键特征：**
- 行业：金融（银行）
- 合同金额：1200万元（大额采购）
- 技术领域：信创基础设施
- 核心挑战：系统兼容性、数据安全、业务连续性
- 合规要求：信创认证、金融监管

**设计意图：** 选择金融行业信创采购场景，因为该场景涉及数据安全、业务连续性、合规监管等多重风险维度，适合作为高难度测试用例。

## 二、风险设计意图

### 2.1 显式埋入的风险（7个）

| 风险ID | 风险点 | 设计意图 |
|--------|--------|----------|
| R001 | 远程维护无数据保护条款 | 测试AI能否识别"服务条款中隐含的数据安全风险"——乙方提供技术支持必然接触甲方数据，但合同未约定保护措施 |
| R002 | 交付时间与验收时间矛盾 | 测试AI能否发现"时间逻辑漏洞"——5个工作日验收分布式存储系统不现实 |
| R003 | 定制固件知识产权归属模糊 | 测试AI能否识别"知识产权归属对金融机构的特殊风险"——仅有使用权但无源代码保障 |
| R004 | 违约金上限过低（5%） | 测试AI能否评估"违约金比例与合同金额的匹配性"——60万上限不足以覆盖1200万合同的潜在损失 |
| R007 | 付款节点与交付节点错位 | 测试AI能否发现"资金风险敞口"——分批交付但一次性支付到货款的问题 |
| R008 | 质保期起算时间不合理 | 测试AI能否识别"质保期实质缩短"的风险——验收周期短导致质保期有效时间减少 |
| R009 | 信创合规条款笼统 | 测试AI能否识别"合规风险"——笼统的信创要求可能无法满足监管检查 |

### 2.2 缺失条款衍生的风险（7个）

| 风险ID | 风险点 | 设计意图 |
|--------|--------|----------|
| R005 | 缺少数据迁移条款 | 测试AI能否识别"信创改造必备条款的缺失"——设备更换必然涉及数据迁移 |
| R006 | 缺少系统集成验收标准 | 测试AI能否识别"验收标准不完整"——仅有设备验收，缺少系统集成验收 |
| R010 | 缺少供应链保障条款 | 测试AI能否识别"供应链风险管理条款缺失"——信创设备芯片供应受国际形势影响 |
| R011 | 缺少间接损失免责条款 | 测试AI能否识别"责任边界不清"——金融机构设备故障的间接损失可能远超设备价值 |
| R012 | 缺少人员管理条款 | 测试AI能否识别"安保管理漏洞"——金融机构数据中心对人员有严格安保要求 |
| R013 | 缺少固件安全更新机制 | 测试AI能否识别"运维安全条款缺失"——信创设备固件更新需要安全审查 |
| R014 | 合同解除后设备处置不明确 | 测试AI能否识别"合同终止条款不完整"——已安装设备的拆除和数据清除责任不清 |

## 三、矛盾设计意图

### 3.1 交付时间与验收时间矛盾（C001）

**设计方法：** 在第四条设定分布式存储60个工作日内交付，在第六条设定5个工作日内完成最终验收。分布式存储系统行业惯例需要30天以上压力测试，两个条款在时间上存在不可调和的矛盾。

**测试价值：** 检测AI能否识别"时间维度的逻辑冲突"，特别是当矛盾涉及专业领域知识（分布式存储验收周期）时。

### 3.2 合同总价与明细清单差异（C002）

**设计方法：** 第三条约定总价1200万元，但第二条设备清单各项合计1140万元，存在60万元差额未说明。

**测试价值：** 检测AI能否进行"数值一致性校验"，发现合同内部的金额矛盾。

### 3.3 知识产权归属与源代码保管矛盾（C003）

**设计方法：** 第十条约定定制固件知识产权归乙方，第七条约定源代码由乙方保管。两个条款单独看都合理，但组合起来导致甲方在特定情况下（乙方停止维护）无法维护核心系统。

**测试价值：** 检测AI能否识别"条款组合产生的隐性矛盾"，特别是需要结合行业背景（金融系统维护需求）才能发现的矛盾。

## 四、缺失条款设计意图

| 缺失ID | 缺失条款 | 设计意图 |
|--------|----------|----------|
| M001 | 数据安全保护条款 | 金融行业采购合同的必备条款，涉及《数据安全法》《个人信息保护法》合规要求 |
| M002 | 数据迁移条款 | 信创改造的必经环节，缺少此条款将导致项目实施无章可循 |
| M003 | 系统集成验收标准 | 信创改造的核心挑战在于系统兼容性，仅有设备验收远远不够 |
| M004 | 供应链保障条款 | 信创设备核心元器件供应受国际形势影响，需要供应链风险管理 |
| M005 | 业务连续性保障条款 | 金融机构核心系统不能中断运行，设备切换需要业务连续性保障 |

**设计原则：** 所有缺失条款均为金融行业IT采购合同的行业惯例必备条款，测试AI是否具备行业知识背景来识别条款缺失。

## 五、测试价值说明

### 5.1 多维度风险识别能力

本测试资产覆盖以下风险维度：
- **数据安全风险：** 远程维护无数据保护（R001）
- **时间逻辑风险：** 交付与验收时间矛盾（C001、R002）
- **知识产权风险：** 定制固件权属不清（R003、C003）
- **财务风险：** 违约金过低（R004）、付款错位（R007）
- **合规风险：** 信创条款笼统（R009）
- **供应链风险：** 缺少供应链保障（R010）
- **业务连续性风险：** 缺少数据迁移（R005）、缺少集成验收（R006）

### 5.2 行业知识检验

本测试资产需要AI具备以下行业知识：
- 分布式存储系统的验收周期（30天以上压力测试）
- 金融行业信创改造的合规要求
- 金融机构数据中心的安保管理要求
- 大额设备采购的付款风险管理

### 5.3 难度评估

**难度等级：hard**

理由：
1. 合同金额大（1200万），条款数量多（17条），信息密度高
2. 风险点涉及专业领域知识（分布式存储、信创合规）
3. 存在隐性矛盾（C003需要组合分析两个条款）
4. 缺失条款需要行业背景知识才能识别
5. 同时存在显式风险和隐式风险，需要不同的检测策略
