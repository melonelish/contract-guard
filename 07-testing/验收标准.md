# Acceptance Criteria

> Version: v1.0 | Last Updated: 2026-06-15

---

## 1. Functional Acceptance

| ID | Acceptance Item | Acceptance Criteria | Status |
|---|---|---|---|
| AC-01 | Contract upload | Supports PDF/Word, drag-and-drop + click upload, progress indicator | |
| AC-02 | OCR recognition | Printed PDF accuracy ≥ 95% | |
| AC-03 | Table restoration | Markdown table format, restoration rate ≥ 90% | |
| AC-04 | Clause analysis | Risk level assessment + legal citations + revision suggestions | |
| AC-05 | Legal citations | All cited legal provisions actually exist in the knowledge base | |
| AC-06 | Case citations | Relevant case retrieval + case number + judgment key points | |
| AC-07 | Cross-validation | Inter-clause conflict detection recall ≥ 80% | |
| AC-08 | Missing clause detection | Detect standard clauses that should exist based on contract type | |
| AC-09 | Report display | All six sections fully displayed, with filtering support | |
| AC-10 | Report export | PDF/Word format export | |
| AC-11 | User registration/login | Email registration + JWT authentication | |
| AC-12 | Permission isolation | Complete data isolation between tenants | |

## 2. Non-Functional Acceptance

| ID | Acceptance Item | Standard |
|---|---|---|
| NF-01 | Review duration | 30-page contract ≤ 15 minutes |
| NF-02 | Concurrency support | 100 TPS |
| NF-03 | First screen load | ≤ 2 seconds |
| NF-04 | System availability | 99.9% (monthly downtime ≤ 43 minutes) |
| NF-05 | Security | TLS 1.3 + AES-256 + No OWASP Top 10 vulnerabilities |
| NF-06 | Hallucination rate | Type A (fabricated laws) 0%, Type B (text tampering) < 1% |

## 3. MVP Minimum Acceptance Criteria

| Grade | Criteria |
|---|---|
| **Blocking (blocks launch)** | AC1-AC4, AC9, AC11, NF01-NF05 all pass |
| **Warning (can defer)** | AC5 (85%), AC7, AC8, AC10, AC12 can be deferred 1-2 weeks |
| **Nice-to-have** | AC6, NF06 can be included in Phase 2 |
