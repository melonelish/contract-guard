# 接口设计

> 版本：v1.0 | 最后更新：2026-06-15

---

## 一、API 设计原则

- RESTful 风格
- 统一 JSON 响应格式
- 分页统一使用 `page` + `page_size`
- 多租户隔离：所有接口自动注入 `tenant_id`

---

## 二、统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "meta": {
    "page": 1,
    "page_size": 20,
    "total": 100
  }
}
```

## 三、核心 API

### 3.1 用户模块

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/auth/register` | 注册 |
| POST | `/api/v1/auth/login` | 登录（返回 JWT） |
| GET | `/api/v1/users/me` | 获取当前用户 |
| PUT | `/api/v1/users/me` | 更新个人信息 |

### 3.2 合同模块

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/contracts/upload` | 上传合同（multipart/form-data） |
| GET | `/api/v1/contracts` | 合同列表（分页） |
| GET | `/api/v1/contracts/{id}` | 合同详情 |
| DELETE | `/api/v1/contracts/{id}` | 删除合同 |

### 3.3 审查模块

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/contracts/{id}/review` | 触发审查 |
| GET | `/api/v1/reviews/{id}` | 获取审查报告 |
| GET | `/api/v1/reviews/{id}/status` | 查询审查进度 |
| GET | `/api/v1/reviews/{id}/export?format=pdf` | 导出报告 |

### 3.4 团队模块

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/teams` | 创建团队 |
| GET | `/api/v1/teams/{id}` | 团队信息 |
| POST | `/api/v1/teams/{id}/members` | 邀请成员 |

### 3.5 编辑模块

| 方法 | 路径 | 说明 |
|---|---|---|
| PUT | `/api/v1/contracts/{id}/draft` | 保存编辑草稿 |
| POST | `/api/v1/contracts/{id}/draft/review` | 重新审查 |
| GET | `/api/v1/contracts/{id}/diff` | 差异对比 |

### 3.6 版本与差异

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v1/contracts/{id}/versions` | 版本历史列表 |
| GET | `/api/v1/contracts/{id}/diffs?from=v1&to=v2` | 版本间差异对比 |

### 3.7 反馈模块

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/reviews/{id}/feedback` | 提交审查反馈（准确/不准确） |

### 3.8 管理后台

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v1/admin/kb/versions` | 知识库版本历史 |

### 3.9 WebSocket

| 端点 | 说明 |
|---|---|
| `ws://host/ws/review/{review_id}` | 审查进度实时推送 |

## 四、接口详情

### 4.1 上传合同

```
POST /api/v1/contracts/upload

Headers:
  Authorization: Bearer {jwt_token}
  Content-Type: multipart/form-data

Body:
  file: (binary)     # PDF/Word/图片文件
  title: "采购合同"   # 可选，不填自动提取

Response 201:
{
  "code": 0,
  "data": {
    "id": "ct-abc123",
    "title": "采购合同",
    "file_url": "https://cdn.xxx.com/contracts/abc123.pdf",
    "status": "uploaded",
    "created_at": "2026-06-15T10:00:00Z"
  }
}
```

### 4.2 触发审查

```
POST /api/v1/contracts/{id}/review

Response 202:
{
  "code": 0,
  "data": {
    "review_id": "rv-xyz789",
    "task_id": "rv-xyz789",
    "status": "queued",
    "estimated_seconds": 900
  }
}
```

### 4.3 WebSocket 推送（渐进式进度）

```
ws://host/ws/review/rv-xyz789

服务端推送事件（按审查阶段逐次推送）:

阶段 1 — 文档解析 (0-15%):
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "parsing", "progress": 5,
   "detail": "PDF 文本提取完成，共 15 页"}
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "parsing", "progress": 10,
   "detail": "识别到 47 条条款"}
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "parsing", "progress": 15,
   "detail": "条款结构化完成"}

阶段 2 — 条款分析 (15-85%):
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "analyzing", "progress": 18,
   "detail": "正在分析条款 5/47", "clause_current": 5, "clause_total": 47}
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "analyzing", "progress": 35,
   "detail": "正在分析条款 20/47", "clause_current": 20}
  ... 每隔 5 条条款推送一次进度 ...
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "analyzing", "progress": 80,
   "detail": "条款分析完成，检测到 12 处风险"}

阶段 3 — 报告生成 (85-95%):
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "reporting", "progress": 90,
   "detail": "正在生成审查报告"}

阶段 4 — 校验 (95-100%):
  {"event": "stage", "review_id": "rv-xyz789", "task_id": "rv-xyz789", "stage": "validating", "progress": 98,
   "detail": "正在校验法条引用..."}

完成:
  {"event": "complete", "review_id": "rv-xyz789", "task_id": "rv-xyz789",
   "summary": {"total_risks": 12, "high": 3, "medium": 5, "low": 4},
   "duration_sec": 842}
```
> **大合同自适应**：条款 > 60 条的合同，`stage` 事件追加预估剩余时间 `"eta_sec": 480`，防止用户焦虑等待。

## 五、统一契约补充

### 5.1 ID 口径

| 字段 | 类型 | 说明 |
|---|---|---|
| `contract_id` | UUID | 合同主键 |
| `review_id` | UUID | 审查主键 |
| `task_id` | UUID | 异步编排任务主键；MVP 与 `review_id` 一一对应 |
| `clause_id` | UUID | 条款主键 |
| `clause_code` | string | 条款展示编号，如 `cl_007` |
| `clause_analysis_id` | UUID | 条款分析主键 |

### 5.2 统一规则

- 对外接口出现条款对象时，同时返回 `clause_id` 和 `clause_code`
- 反馈接口绑定 `clause_analysis_id`，不直接绑定 `clause_id`
- WebSocket 使用 `review_id` 订阅，事件体携带 `task_id`
