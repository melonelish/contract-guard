## 二、核心表 DDL

### 2.1 tenants（租户表）

```sql
CREATE TABLE tenants (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    plan VARCHAR(20) NOT NULL DEFAULT 'free',
    created_at TIMESTAMP DEFAULT NOW()
);

-- RLS 策略：每个租户只看自己的数据
ALTER TABLE tenants ENABLE ROW LEVEL SECURITY;
```

### 2.2 users（用户表）

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id),
    email VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100),
    role VARCHAR(20) NOT NULL DEFAULT 'member',
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_users_tenant ON users(tenant_id);
CREATE INDEX idx_users_email ON users(email);
```

### 2.3 contracts（合同表）

```sql
CREATE TABLE contracts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL REFERENCES users(id),
    title VARCHAR(500),
    file_url VARCHAR(1000) NOT NULL,
    file_type VARCHAR(10) NOT NULL,
    file_size BIGINT,
    page_count INT,
    contract_type VARCHAR(50),
    status VARCHAR(20) DEFAULT 'uploaded',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_contracts_tenant ON contracts(tenant_id);
CREATE INDEX idx_contracts_user ON contracts(user_id);
```

### 2.4 reviews（审查记录表）

```sql
CREATE TABLE reviews (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id UUID NOT NULL REFERENCES contracts(id) ON DELETE CASCADE,
    status VARCHAR(20) DEFAULT 'pending',
    report_json JSONB,
    total_risks INT,
    high_risks INT,
    medium_risks INT,
    low_risks INT,
    duration_ms INT,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_reviews_contract ON reviews(contract_id);
CREATE INDEX idx_reviews_status ON reviews(status);
```

### 2.5 clauses（条款表）

```sql
CREATE TABLE clauses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id UUID NOT NULL REFERENCES contracts(id) ON DELETE CASCADE,
    title VARCHAR(200),
    category VARCHAR(50),
    page_num INT,
    line_start INT,
    line_end INT,
    full_text TEXT NOT NULL,
    contains_table BOOLEAN DEFAULT FALSE,
    table_markdown TEXT,
    embedding VECTOR(1024)  -- bge-m3 embedding for clause-level similarity search
);

CREATE INDEX idx_clauses_contract ON clauses(contract_id);
CREATE INDEX idx_clauses_embedding ON clauses USING ivfflat (embedding vector_cosine_ops);
```

### 2.6 clause_analyses（条款分析表）

```sql
CREATE TABLE clause_analyses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    review_id UUID NOT NULL REFERENCES reviews(id) ON DELETE CASCADE,
    clause_id UUID NOT NULL REFERENCES clauses(id),
    risk_level VARCHAR(10),
    risk_category VARCHAR(50),
    analysis_json JSONB NOT NULL,
    confidence DECIMAL(3,2),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_analyses_review ON clause_analyses(review_id);
CREATE INDEX idx_analyses_risk ON clause_analyses(risk_level);
```

### 2.7 audit_logs（审计日志表）

```sql
CREATE TABLE audit_logs (
    id BIGSERIAL PRIMARY KEY,
    tenant_id UUID NOT NULL,
    user_id UUID,
    action VARCHAR(100) NOT NULL,
    resource_type VARCHAR(50),
    resource_id UUID,
    details JSONB,
    ip_address INET,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_audit_tenant ON audit_logs(tenant_id);
CREATE INDEX idx_audit_time ON audit_logs(created_at DESC);
```

## 三、RAG 知识库表

### 3.1 law_articles（法律条文表）

```sql
CREATE TABLE law_articles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    law_name VARCHAR(200) NOT NULL,
    article_number VARCHAR(50) NOT NULL,
    article_title VARCHAR(300),
    full_text TEXT NOT NULL,
    keywords TEXT[],
    effective_date DATE,
    status VARCHAR(20) DEFAULT '现行有效',
    embedding VECTOR(1024),
    UNIQUE(law_name, article_number)
);

CREATE INDEX idx_law_embedding ON law_articles USING ivfflat (embedding vector_cosine_ops);
```

### 3.2 legal_cases（判例表）

```sql
CREATE TABLE legal_cases (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    case_number VARCHAR(100) UNIQUE NOT NULL,
    court_level VARCHAR(50),
    court_name VARCHAR(200),
    dispute_focus TEXT,
    judgment_point TEXT,
    related_laws TEXT[],
    embedding VECTOR(1024)
);
```

## 四、索引策略

| 索引 | 用途 | 类型 |
|---|---|---|
| `idx_contracts_tenant` | 租户合同列表 | B-Tree |
| `idx_reviews_status` | 审查状态筛选 | B-Tree |
| `idx_clauses_contract` | 条款查询 | B-Tree |
| `idx_analyses_risk` | 按风险等级筛选 | B-Tree |
| `idx_audit_time` | 审计日志时间查询 | B-Tree DESC |
| `idx_law_embedding` | 法条语义检索 | IVFFlat |
| `idx_case_embedding` | 判例语义检索 | IVFFlat |
| `idx_contracts_text` | 合同内全文搜索 | GIN (tsvector) |

## 数据库迁移方案

### Alembic 配置

```bash
# backend/alembic.ini
[alembic]
script_location = backend/migrations
sqlalchemy.url = postgresql://contractguard:changeme@localhost:5432/contractguard

# backend/migrations/env.py
# 使用 SQLAlchemy MetaData 自动检测模型变更
target_metadata = Base.metadata
```

### 迁移纪律

| 规则 | 说明 |
|---|---|
| 每次 schema 变更 = 一个新 migration | `alembic revision --autogenerate -m "add_clauses_embedding"` |
| migration 必须可逆 | 每个 `upgrade()` 必须有对应的 `downgrade()` |
| CI 自动校验 | PR 合并前运行 `alembic upgrade head`，验证无冲突 |
| 生产环境先备份 | `pg_dump` > 运行 migration > 验证 > 回滚预案 |
| 回滚演练 | 每次发布前在 staging 环境执行 upgrade → downgrade → upgrade |

### 示例迁移

```bash
# 创建迁移（本例：添加 clauses.embedding 列）
alembic revision --autogenerate -m "add_embedding_to_clauses"

# 升级到最新
alembic upgrade head

# 回滚到上一个版本
alembic downgrade -1

# 查看当前版本
alembic current
```
