## 二、认证与授权

### 2.1 JWT 认证流程

```
用户登录 → 验证邮箱/密码 → 生成 JWT (有效期 24h)
                              │
                              ├── Access Token: 24h
                              └── Refresh Token: 7d

API 请求：Authorization: Bearer {access_token}
Token 过期 → 使用 Refresh Token 换新 Access Token
```

### 2.2 RBAC 权限模型

| 角色 | 权限范围 |
|---|---|
| `admin` | 管理租户、查看所有数据、配置规则 |
| `manager` | 管理本团队成员、查看团队审查统计 |
| `member` | 上传合同、查看自己的审查报告 |
| `viewer` | 只读查看被分享的报告 |

## 三、数据安全

### 3.1 合同文件加密

```python
# 存储前加密
encrypted = AES256_GCM.encrypt(
    plaintext=file_content,
    key=tenant_encryption_key,
    nonce=os.urandom(12)
)

# 读取时解密（仅授权用户）
plaintext = AES256_GCM.decrypt(
    ciphertext=encrypted,
    key=tenant_encryption_key,
    nonce=nonce
)
```

### 3.2 敏感信息脱敏

在存储前自动检测并脱敏：

| 数据类型 | 检测正则 | 脱敏方式 |
|---|---|---|
| 身份证号 | `\d{17}[\dXx]` | 保留前3后4 |
| 手机号 | `1[3-9]\d{9}` | 保留前3后4 |
| 银行卡号 | `\d{16,19}` | 保留后4位 |
| 统一信用代码 | `[0-9A-HJ-NPQRTUWXY]{18}` | 公开信息，不脱敏 |

## 四、LLM 安全

### 4.1 Prompt 注入防护

```
输入层防护：
  · 合同文件在上传时扫描恶意文本
  · 用户自定义规则需要经过 HTML 转义
  · 所有用户输入的提示词长度限制 500 字符

输出层防护：
  · Validator Agent 对所有输出进行内容审查
  · 检测并移除可能包含的 Prompt 注入残留
  · 严禁 LLM 输出中包含系统提示词原文
```

### 4.2 模型输出过滤

| 风险 | 过滤方式 |
|---|---|
| 越权法律意见 | 检测"应起诉""已构成犯罪"等结论性表述，替换为"建议咨询律师" |
| 用户数据泄露 | 输出中不应包含其他用户的合同内容 |
| 有害内容 | 通用内容安全 API 过滤 |

## 五、审计日志

### 5.1 记录内容

| 事件 | 记录字段 |
|---|---|
| 用户登录/登出 | user_id, ip, 时间 |
| 合同上传 | user_id, contract_id, 文件名, 文件大小 |
| 触发审查 | user_id, contract_id, review_id |
| 查看报告 | user_id, review_id, 查看时间 |
| 导出报告 | user_id, review_id, 导出格式 |

### 5.2 不可篡改

```sql
-- 审计表使用 WAL（Write-Ahead Log）+ 定期归档
-- 禁止 DELETE/UPDATE 操作
CREATE POLICY no_delete ON audit_logs
    FOR DELETE USING (false);

CREATE POLICY no_update ON audit_logs
    FOR UPDATE USING (false);
```
