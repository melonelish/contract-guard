## 2. Authentication & Authorization

### 2.1 JWT Authentication Flow

```
User Login → Verify email/password → Generate JWT (valid 24h)
                              │
                              ├── Access Token: 24h
                              └── Refresh Token: 7d

API Request: Authorization: Bearer {access_token}
Token Expired → Use Refresh Token to exchange for new Access Token
```

### 2.2 RBAC Permission Model

| Role | Permission Scope |
|---|---|
| `admin` | Manage tenants, view all data, configure rules |
| `manager` | Manage team members, view team review statistics |
| `member` | Upload contracts, view own review reports |
| `viewer` | Read-only access to shared reports |

## 3. Data Security

### 3.1 Contract File Encryption

```python
# Encrypt before storage
encrypted = AES256_GCM.encrypt(
    plaintext=file_content,
    key=tenant_encryption_key,
    nonce=os.urandom(12)
)

# Decrypt on read (authorized users only)
plaintext = AES256_GCM.decrypt(
    ciphertext=encrypted,
    key=tenant_encryption_key,
    nonce=nonce
)
```

### 3.2 Sensitive Information Masking

Auto-detect and mask before storage:

| Data Type | Detection Regex | Masking Method |
|---|---|---|
| ID Card Number | `\d{17}[\dXx]` | Keep first 3, last 4 |
| Phone Number | `1[3-9]\d{9}` | Keep first 3, last 4 |
| Bank Card Number | `\d{16,19}` | Keep last 4 digits |
| Unified Social Credit Code | `[0-9A-HJ-NPQRTUWXY]{18}` | Public info, do not mask |

## 4. LLM Security

### 4.1 Prompt Injection Protection

```
Input Layer Protection:
  · Contract files scanned for malicious text on upload
  · User-defined rules undergo HTML escaping
  · All user input prompts limited to 500 characters

Output Layer Protection:
  · Validator Agent performs content review on all outputs
  · Detect and remove any Prompt injection residue
  · Strictly forbid LLM output containing system prompt text
```

### 4.2 Model Output Filtering

| Risk | Filtering Method |
|---|---|
| Unauthorized legal opinions | Detect conclusive language like "should sue" or "constitutes a crime"; replace with "recommend consulting a lawyer" |
| User data leakage | Output must not contain other users' contract content |
| Harmful content | General content safety API filtering |

## 5. Audit Logging

### 5.1 Logged Events

| Event | Logged Fields |
|---|---|
| User login/logout | user_id, ip, timestamp |
| Contract upload | user_id, contract_id, filename, file size |
| Trigger review | user_id, contract_id, review_id |
| View report | user_id, review_id, view timestamp |
| Export report | user_id, review_id, export format |

### 5.2 Immutability

```sql
-- Audit table uses WAL (Write-Ahead Log) + periodic archiving
-- DELETE/UPDATE operations are forbidden
CREATE POLICY no_delete ON audit_logs
    FOR DELETE USING (false);

CREATE POLICY no_update ON audit_logs
    FOR UPDATE USING (false);
```
