## 二、技术选型决策

| 层级 | 技术 | 选型理由 |
|---|---|---|
| **前端** | React 18 + TypeScript + Ant Design | 组件生态丰富，中文支持好；微信小程序用 Taro 跨端 |
| **API Gateway** | Kong / Nginx | 生产级网关，支持限流、认证插件 |
| **后端框架** | Python FastAPI | 异步高性能，AI 生态最好（LangChain/LlamaIndex均原生Python） |
| **Agent 框架** | 自研（不依赖 LangChain） | 精确控制 Agent 通信和重试逻辑，避免框架黑盒 |
| **LLM 调用** | OpenAI SDK 兼容 + 多模型适配层 | 支持 MiMo 2.5 / DeepSeek V4-Flash 热切换 |
| **消息队列** | Redis Streams | 轻量，无需额外部署 RabbitMQ/Kafka |
| **数据库** | PostgreSQL 15 | 成熟稳定，支持 JSONB 和向量扩展 pgvector |
| **向量数据库** | Milvus（生产）/ pgvector（开发） | 支持多维过滤 + 混合检索 |
| **文件存储** | MinIO（兼容 S3） | 私有化部署友好，兼容 S3 API |
| **关键词检索** | Elasticsearch 8.x | 中文分词 + BM25 检索 |
| **部署** | Docker Compose（开发）→ K8s（生产） | 渐进式架构 |
| **监控** | Prometheus + Grafana + LangSmith | 全链路可观测 |

## 三、核心数据流

```
用户上传合同 PDF
      │
      ▼
┌─────────────┐
│ API Gateway  │ ← 认证 + 限流
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 合同服务      │ ← 保存文件到 MinIO
│              │ ← 写入合同记录到 PostgreSQL
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│ 审查编排引擎      │ ← 创建审查任务
│ (Orchestrator)  │ ← 发送任务到 Redis 队列
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│ Supervisor Agent│ ← 从 Redis 取任务
│                 │ ← 协调子 Agent 执行
└──────┬──────────┘
       │
  ┌────┴────┬────────┐
  ▼         ▼        ▼
Parser → Analyzer → Report → Validator
                      │
                      ▼
              ┌──────────────┐
              │ 保存结果到 DB │
              │ 返回 report_id │
              └──────┬───────┘
                     │
                     ▼
              ┌──────────────┐
              │ WebSocket通知 │ ← 前端收到 → 展示报告
              │ 前端审查完成   │
              └──────────────┘
```

## 四、安全架构

### 4.1 数据安全环

```
 网络层      TLS 1.3 全程加密
     │
 应用层      JWT 认证 + RBAC 权限
     │
 数据层      AES-256-at-rest 加密存储
     │        合同原文加密、脱敏数据单独存储
     │
 审计层      全操作日志、不可删除、不可篡改
```

### 4.2 多租户隔离

```
           ┌────────────┐
           │   Shared    │ ← 法律法规库（所有租户共享，只读）
           │   Layer     │
           └─────┬──────┘
                 │
     ┌───────────┼───────────┐
     │           │           │
┌────▼────┐ ┌────▼────┐ ┌────▼────┐
│ Tenant  │ │ Tenant  │ │ Tenant  │
│    A    │ │    B    │ │    C    │
│         │ │         │ │         │
│ 合同库   │ │ 合同库   │ │ 合同库   │
│ 审查记录 │ │ 审查记录 │ │ 审查记录 │
│ 自定义   │ │ 自定义   │ │ 自定义   │
│ 规则     │ │ 规则     │ │ 规则     │
└─────────┘ └─────────┘ └─────────┘
```

SQL 查询自动注入 `WHERE tenant_id = {current_tenant}`，数据库行级安全策略兜底。
