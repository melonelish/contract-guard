## 一、Phase 1 — MVP（M1-M3, 12周）

### Sprint 划分

| Sprint | 时间 | 目标 | 主要任务 |
|---|---|---|---|
| Sprint 1 | W1-W2 | 技术选型 + 环境搭建 | 后端框架搭建、数据库设计、CI/CD 流水线 |
| Sprint 2 | W3-W4 | Parser Agent | PDF解析、OCR集成、表格还原 |
| Sprint 3 | W5-W6 | Analyzer Agent | 条款分析链路、RAG知识库搭建、法条检索 |
| Sprint 4 | W7-W8 | Report + Validator | 报告生成、校验链路、WebSocket推送 |
| Sprint 5 | W9-W10 | 前端开发 | 上传页面、报告展示页面、用户系统 |
| Sprint 6 | W11-W12 | 联调 + 测试 + 修复 | 端到端测试、性能优化、Bug修复 |

## 四、风险管理

| 风险 | 概率 | 等级 | 应对 |
|---|---|---|---|
| LLM API 不稳定 | 中 | 高 | 多模型热切换、本地备选模型 |
| Agent 准确率不达标 | 中 | 高 | 持续评测、法务顾问反馈闭环 |
| 竞品提前发布 | 高 | 中 | 加速 MVP，先占市场 |
| 法律合规风险 | 低 | 高 | 法务顾问全程参与、免责声明硬约束 |
| 团队成员离职 | 低 | 中 | 文档全、知识分散，非单点依赖 |
