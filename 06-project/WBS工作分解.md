# WBS 工作分解结构

> 版本：v1.0 | 最后更新：2026-06-15

---

## Phase 1 MVP 工作分解

```
1. 项目启动
   1.1 环境搭建
       1.1.1 云资源申请（服务器、数据库、存储）
       1.1.2 CI/CD 流水线配置
       1.1.3 域名 + SSL 证书配置
   1.2 技术调研
       1.2.1 LLM 模型性能评测
       1.2.2 Embedding 模型选型评测
       1.2.3 OCR 方案选型评测
   1.3 数据库设计
       1.3.1 ER 图设计
       1.3.2 DDL 编写与评审
       1.3.3 数据迁移脚本

2. 后端服务开发
   2.1 用户模块
       2.1.1 注册/登录 API
       2.1.2 JWT 认证中间件
       2.1.3 多租户隔离中间件
   2.2 合同模块
       2.2.1 文件上传（MinIO）
       2.2.2 合同列表/详情/删除 API
       2.2.3 合同类型自动识别
   2.3 审查模块
       2.3.1 审查编排引擎
       2.3.2 审查任务队列
       2.3.3 WebSocket 进度推送
       2.3.4 报告导出（PDF/Word）

3. Agent 引擎开发
   3.1 Parser Agent
       3.1.1 PDF 文本提取
       3.1.2 表格检测与 Markdown 还原
       3.1.3 OCR 集成（扫描件）
       3.1.4 条款结构化输出
   3.2 Analyzer Agent
       3.2.1 单条款分析 Prompt
       3.2.2 RAG 知识库搭建（法律法规）
       3.2.3 RAG 知识库搭建（判例库）
       3.2.4 混合检索引擎
       3.2.5 交叉校验逻辑
       3.2.6 缺失项检测逻辑
   3.3 Report Agent
       3.3.1 报告模板设计
       3.3.2 报告生成 Prompt
   3.4 Validator Agent
       3.4.1 法条存在性校验
       3.4.2 原文一致性校验
       3.4.3 审核不通过重试机制
   3.5 Drafter Agent（合同起草）
       3.5.1 起草 Prompt 设计
       3.5.2 合同范本 RAG 知识库搭建
       3.5.3 Annotation 标注逻辑
       3.5.4 起草→审查管道衔接
       3.5.5 起草结果用户交互（确认/修改/拒绝条款）

4. 前端开发
   4.1 基础框架
       4.1.1 React 项目初始化
       4.1.2 路由 + 状态管理
       4.1.3 主题定制
   4.2 用户端页面
       4.2.1 登录/注册页
       4.2.2 合同上传页
       4.2.3 合同列表页
       4.2.4 审查报告页
       4.2.5 报告导出

5. 测试
   5.1 单元测试
   5.2 集成测试
   5.3 Agent 评测
   5.4 安全测试
   5.5 性能测试

6. 部署上线
   6.1 Docker 镜像构建
   6.2 K8s 部署配置
   6.3 监控 + 告警配置
   6.4 灰度发布验证
```
