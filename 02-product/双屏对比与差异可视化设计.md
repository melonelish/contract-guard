## 一、功能定位

**核心目标**：让用户一眼看懂"AI 要我改哪里"，而不是读完审查报告再手动找位置改。

**价值主张**：
- 降低用户理解成本（不用脑补修改效果）
- 提升修改效率（一键接受/批量处理）
- 增强信任感（所见即所得，改哪看得见）

### 2.2 两种视图模式

| 模式 | 适用场景 | 界面特征 |
|---|---|---|
| **双屏模式** | 大幅修改、需要对照原文 | 左右分栏，左侧原文，右侧修改后，中间无交互 |
| **单屏修订模式** | 小幅修改、快速确认 | 单栏显示，用删除线+高亮标注修改处，类似 Word 修订 |

**切换方式**：右上角"双屏模式"开关，一键切换。

### 4.2 操作按钮

| 操作 | 效果 | 快捷键 |
|---|---|---|
| **接受** | 将该处修改应用到当前合同 | Space |
| **手动改** | 弹出编辑框，用户自行调整 | Enter |
| **忽略** | 保留原文，不采纳 AI 建议 | Delete |
| **定位** | 双屏自动滚动到该条款位置 | Click |
| **一键接受** | 批量接受所有未处理的建议 | Ctrl+A |

### 4.3 一键接受后的流程

```
用户点击 [一键接受所有建议]
        │
        ▼
系统生成新版合同文档
        │
        ▼
弹出确认："已生成修改后版本，是否下载？"
        │
        ├── [下载新合同] → Word/PDF 导出
        ├── [继续审查]   → 回到差异列表，继续处理剩余项
        └── [重新审查]   → 提交修改后合同，开启新一轮审查
```

### 5.2 Prompt 改造（审查 Agent）

在原有 System Prompt 基础上，增加输出格式要求：

```yaml
# 新增要求：结构化差异输出

除了原有分析外，你需要输出该条款的 inline_diff 数组：

inline_diff 格式：
- type: "unchanged" | "deleted" | "added"
- text: "具体文本内容"

示例：
原始条款："验收时间为15个工作日"
建议改为："验收时间为3个工作日"

inline_diff 应为：
[
  {"type": "unchanged", "text": "验收时间为"},
  {"type": "deleted", "text": "15"},
  {"type": "added", "text": "3"},
  {"type": "unchanged", "text": "个工作日"}
]

要求：
1. 尽可能精准定位到字词级别的差异
2. unchanged 段落不宜过长（建议不超过20字）
3. 如果整句都是新增或删除，用一个 diff 项表示
```

## 七、内置编辑器

### 7.1 为什么必须内置编辑器

```
外部修改的痛点：
───────────────
用户下载 Word → 用 WPS 打开 → 表格错位 → 段落乱掉 →
改完再上传审查 → 系统不认识这是改过的版本 → 丢失审查上下文

内置编辑器的价值：
───────────────
用户看到审查建议 → 直接在系统里改 → 格式永远不乱 →
改完点"重新审查" → 系统知道改了哪里 → 自动对比差异 →
用户始终留在系统内，不会流失到 Word/外部工具
```

### 7.3 AI 标注嵌入编辑器

核心设计：AI 风险标记以 `<mark>` 标签嵌入编辑器内容区内，不可被用户删除。

```html
<!-- 编辑器中嵌入的条款 -->
<div class="clause-block" data-clause-id="cl_007">
  <div class="clause-content" contenteditable="true">
    合同签订后，甲方向乙方支付合同总价款的
    <mark class="risk-high" 
          data-risk-id="R-001"
          contenteditable="false">30%</mark>
    作为预付款。
  </div>
  
  <!-- AI 建议浮层（点击标记时展开） -->
  <div class="ai-suggestion-banner">
    <div class="suggestion-header">
      🔴 高风险 — 违约金比例可能过高
    </div>
    <div class="suggestion-body">
      建议改为 <strong>20%</strong>，且不超过实际损失
    </div>
    <div class="suggestion-actions">
      <button class="btn-accept" 
              onclick="acceptSuggestion('R-001')">应用此建议</button>
      <button class="btn-dismiss"
              onclick="dismissSuggestion('R-001')">✕</button>
    </div>
    <div class="suggestion-source">
      📚 依据：《民法典》第585条
    </div>
  </div>
</div>
```

**规则**：`<mark contenteditable="false">` 确保用户编辑周围文字时不会误删 AI 标注。只能通过"应用建议"或"忽略"来消除标记。

### 7.4 "应用此建议"的交互

```
用户点击 [应用此建议]
        │
        ▼
编辑器内：标记<mark>30%</mark> → 替换为 → <mark class="resolved">20%</mark>
                                        ↑
                                   颜色从红变绿（表示已修改）
        │
        ▼
AI 建议浮层收起，该条款从"待处理"列表中移除
        │
        ▼
进度条更新：4/5 已处理
```

### 7.5 工具栏操作

| 操作 | 效果 | 快捷键 |
|---|---|---|
| **接受当前建议** | 应用当前展开的那条 AI 建议 | Ctrl+Enter |
| **接受全部建议** | 批量应用所有未处理的 AI 建议 | Ctrl+Shift+A |
| **显示/隐藏 AI 建议** | 切换编辑器内所有 `<mark>` 标签的可见性 | — |
| **保存草稿** | 保存当前编辑进度（不触发审查） | Ctrl+S |
| **重新审查** | 保存修改 → 自动提交审查 → 回到审查报告页 | Ctrl+R |
| **撤销/重做** | 编辑操作回退 | Ctrl+Z / Ctrl+Y |

### 7.6 修改追踪与增量审查

```python
# 用户修改后点击"重新审查"，系统精准知道改了什么
class ContractRevision:
    """每次保存生成一个快照"""
    
    def save_snapshot(self, contract_id: str, content: dict):
        previous = self.get_latest_snapshot(contract_id)
        
        # 对比前后两个版本
        changed_clauses = diff(
            old=previous.clauses,
            new=content.clauses
        )
        
        snapshot = {
            "version": previous.version + 1,
            "content": content,
            "changed_clauses": changed_clauses,  # 只改了三条
            "timestamp": now(),
        }
        
        return snapshot
    
    def incremental_review(self, contract_id: str):
        """增量审查：只审查修改过的条款"""
        latest = self.get_latest_snapshot(contract_id)
        changed = latest.changed_clauses  # 只有 3 条
        
        # 只对变更的条款重新分析
        results = []
        for clause_id in changed:
            clause = latest.content.clauses[clause_id]
            analysis = analyzer_agent.analyze(clause)
            results.append(analysis)
        
        # 未修改的条款保留原审查结果
        previous_review = self.get_review(contract_id)
        merged = merge_results(previous_review, results)
        
        return merged
```

**增量审查的价值**：
- 用户改完 3 条，系统只重新审查这 3 条，不是全合同重来
- 速度从 30 秒降到 5 秒
- Token 消耗从 ¥2.35 降到 ¥0.3
