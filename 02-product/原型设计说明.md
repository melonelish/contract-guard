# 原型设计说明

> 版本：v1.0 | 最后更新：2026-06-15

---

## 一、核心页面清单

| 页面 | 路由 | 用户操作 |
|---|---|---|
| 首页/落地页 | `/` | 产品介绍、CTA 引导注册 |
| 登录注册 | `/login` `/register` | 邮箱/微信登录 |
| 合同上传 | `/upload` | 拖拽上传 PDF/Word、查看上传进度 |
| 审查报告（单屏） | `/review/:id` | 左侧合同原文+右侧风险列表，逐条查看建议 |
| 内置编辑器 | `/review/:id/edit` | 在线修改合同，AI建议嵌入，一键应用 |
| 打印弹窗 | 模态框 | 选择版本、纸张格式、预览打印 |
| 合同列表 | `/contracts` | 查看/搜索/筛选历史合同 |
| 审查历史 | `/reviews` | 查看历次审查记录 |
| 账户管理 | `/account` | 查看余额、购买次数、API配置（企业版） |

---

## 二、核心交互流程

详见 **《双屏对比与差异可视化设计.md》第九章：完整闭环流程**。

简要概括：
1. 上传合同 → AI 审查 → 左侧原文 + 右侧风险列表
2. 进入编辑模式 → 风险标注嵌入编辑器 → 一键应用/手动修改
3. 重新审查 → 增量分析变更条款
4. 满意 → 导出/打印（三种版本可选）

---

## 三、设计原则

- **所见即所得**：审查建议直接嵌入合同文本，不需要对照两份文档
- **不让用户离开系统**：编辑、审查、打印全在线完成，无需下载到本地
- **移动端适配**：微信小程序端主要支持上传和查看，编辑在 PC 端完成
