## 二、P1 — 小微企业创始人

### 基本信息

| 项 | 内容 |
|---|---|
| 姓名代表 | 张总，32岁 |
| 公司 | 某科技公司，25人 |
| 角色 | 创始人兼CEO |
| 收入 | 年薪50万+ |
| 教育 | 本科（计算机专业） |

### 典型一天

```
09:00  到公司，看邮件
10:00  供应商发来一份15页的采购合同需要签署
10:05  打开合同，翻到第3页...付款条件...嗯...
10:20  翻到第8页...知识产权条款...这是什么意思？
10:35  放弃阅读，微信问律师朋友"帮我看下这份合同"
11:00  律师朋友："我手头有案子，晚上帮你看"
18:00  律师朋友说没什么大问题
18:30  签字发回去了
```

**痛点在 10:05-10:35 那 30 分钟：他花了半小时读合同，但什么结论也没得到。**

### 核心需求

- 三五分钟内知道合同有没有重大风险
- 看不懂的地方有人翻译成大白话
- 相信 AI 的判断（有法条依据）
- 价格不贵（< 200 元/月可以冲动消费）

### 使用场景

| 场景 | 频率 | 合同类型 |
|---|---|---|
| 签新供应商合同 | 每月 2-3 次 | 采购/服务合同 |
| 签客户合同 | 每月 3-5 次 | 销售合同/服务协议 |
| 签合作协议 | 每季度 1-2 次 | 战略合作/代理协议 |
| 员工入职签合同 | 每月 1-3 次 | 劳动合同 |

## 三、P2 — 企业法务

### 基本信息

| 项 | 内容 |
|---|---|
| 姓名代表 | 李法务，28岁 |
| 公司 | 某制造企业，300人 |
| 角色 | 法务专员（全公司就他一个法务） |
| 收入 | 月薪 1.5万 |
| 教育 | 法学本科，通过司法考试 |

### 典型一天

```
08:30  采购部发来三份合同，要求下班前给意见
09:00  HR发来新员工劳动合同，要求审核
09:30  业务部催："上周那份合同审好了吗？对方在催"
10:00  开始审第1份 — 30分钟
10:30  开始审第2份 — 25分钟
11:00  开始审第3份 — 40分钟（这份条款比较复杂）
11:40  回复采购部意见
12:00  午饭
13:30  HR那份合同审了15分钟（劳动合同模板化，快）
14:00  业务部那份合同，涉及海外条款，需要查资料...
16:00  查完了，出意见
17:00  写月度合同审查统计报告
18:00  下班（今天算是早的）
```

**痛点：每天 50% 时间花在重复性的基础审查上，真正需要深度思考的复杂合同反而没时间认真看。**

### 核心需求

- 基础审查自动化，留出精力做复杂合同
- 审查标准统一（新人来了能快速上手）
- 能标记"同意"和"有争议"（内部协作）
- 数据统计：每月审了多少、什么类型的、风险分布

## 四、P3 — 律所律师

### 基本信息

| 项 | 内容 |
|---|---|
| 姓名代表 | 王律师，35岁 |
| 律所 | 中型律所，30人 |
| 角色 | 合伙人 |
| 收入 | 年薪 80万+ |

### 典型场景

客户委托审查一份融资合同，王律师需要：
1. 通读全文，标注风险点
2. 逐条分析法律依据
3. 检索相似判例
4. 撰写审查意见书
5. 与客户开会讲解

传统流程：4-6 小时（其中基础工作占 60%）

### 核心需求

- 不是替代自己，而是把 60% 基础工作自动化
- 生成的报告可以直接作为意见书初稿
- 支持私有化部署（客户合同不能上传公共云）
