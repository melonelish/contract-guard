# 前端页面状态机与异常流

> 版本：v1.0 | 最后更新：2026-06-16

---

## 一、文档目的

本文件用于固定前端主要页面的状态、空态、异常态和恢复动作。

目标：

- 防止 Agent 只做“正常 happy path”
- 防止不同页面对同一种错误做出不一致处理
- 让前端实现可以直接按状态机搭页面

---

## 二、全局页面状态约定

每个页面至少支持以下基础状态：

- `idle`
- `loading`
- `success`
- `empty`
- `error`
- `forbidden`

页面级错误展示优先级：

1. `unauthorized`
2. `forbidden`
3. `not_found`
4. `conflict`
5. `retryable_error`
6. `generic_error`

## 三、合同列表页 `/contracts`

### 3.1 状态机

```text
idle
  -> loading
  -> success
  -> empty
  -> error
```

### 3.2 状态定义

| 状态 | 条件 | UI |
|---|---|---|
| `loading` | 首次进入或切换筛选 | 骨架屏 6 行 |
| `success` | 返回 `items.length > 0` | 表格/卡片列表 |
| `empty` | 返回 `items.length = 0` | 空态文案 + 上传按钮 |
| `error` | 请求失败 | 错误提示 + 重试按钮 |
| `forbidden` | 403 | 无权限态 |

### 3.3 空态文案

`你还没有上传过合同，先上传一份开始审查。`

## 四、上传页 `/upload`

### 4.1 状态机

```text
idle
  -> validating
  -> uploading
  -> uploaded
  -> review_queued
  -> error
```

### 4.2 异常流

| 场景 | 错误码 | UI 动作 |
|---|---|---|
| 文件格式不支持 | `3004` | 就地提示，不发请求 |
| 文件超过 50MB | `3005` | 就地提示，不发请求 |
| 上传失败 | `1007` | 保留文件名，允许重试 |
| 触发审查失败 | `4004` | 提示 AI 服务不可用，允许稍后重试 |

### 4.3 成功跳转

- 上传成功且自动触发审查后：
  跳转 `/review/:review_id`
- 若审查仍在排队：
  默认展示进度页，不先回列表

## 五、审查详情页 `/review/:id`

### 5.1 状态机

```text
loading
  -> queued
  -> parsing
  -> analyzing
  -> reporting
  -> validating
  -> completed
  -> failed
  -> not_found
  -> forbidden
```

### 5.2 页面规则

| 状态 | UI |
|---|---|
| `queued` | 排队提示 + 预计等待时间 |
| `parsing` | 进度条 + “正在解析合同” |
| `analyzing` | 进度条 + 当前条款进度 |
| `reporting` | 文案提示 + 禁止操作按钮 |
| `validating` | 文案提示 + 禁止操作按钮 |
| `completed` | 左侧合同 + 右侧风险列表 |
| `failed` | 失败说明 + 重试按钮 |

### 5.3 异常流

| 场景 | 错误码 | 动作 |
|---|---|---|
| 审查不存在 | `4001` | 404 页面 |
| 无权限查看 | `1003` | 无权限态 |
| 审查失败 | `4003` | 显示“重新审查” |
| WebSocket 票据过期 | 关闭码 `4001` | 静默重新申请一次 ticket |
| WebSocket 再次失败 | `4500` 或未知 | 回退到轮询 `/status` |

### 5.4 降级规则

- WebSocket 不可用时，不阻断页面使用
- 页面自动切换到轮询模式，每 5 秒请求一次状态接口

## 六、编辑页 `/review/:id/edit`

### 6.1 状态机

```text
loading
  -> editing
  -> saving
  -> save_success
  -> save_conflict
  -> re_reviewing
  -> re_review_failed
```

### 6.2 关键异常流

| 场景 | 错误码 | 动作 |
|---|---|---|
| 草稿版本冲突 | `3006` | 弹窗提示刷新 |
| 重新审查进行中 | `4002` | 跳转对应审查页 |
| 重新审查失败 | `4003` | 保留草稿，不丢编辑内容 |

### 6.3 保存原则

- `Ctrl+S` 仅保存草稿，不触发重审
- `重新审查` 必须先保存成功，再触发重审
- 任何失败都不能清空编辑器内容

## 七、登录页 `/login`

### 7.1 状态机

```text
idle
  -> submitting
  -> success
  -> invalid_credentials
  -> error
```

### 7.2 异常流

| 场景 | 错误码 | 动作 |
|---|---|---|
| 邮箱/密码错误 | `2001` | 表单内提示 |
| Token 过期 | `2002` | 清本地会话，重新登录 |
| 服务错误 | `1007` | 顶部错误条 |

## 八、前端组件级空态/错误态

### 8.1 风险列表

- 空态：`未检测到风险项`
- 错误态：`风险列表加载失败，请重试`

### 8.2 版本历史

- 空态：`当前合同还没有历史版本`
- 错误态：`版本记录获取失败`

### 8.3 差异对比

- 空态：`两个版本之间没有检测到差异`
- 错误态：`差异计算失败，请稍后重试`

---

## 九、实现要求

- 页面状态禁止只用一个 `loading:boolean` 表示
- 对于审查页和编辑页，必须使用显式枚举状态
- 前端文案判断以 `code` 为准，不直接依赖后端 `message`
- WebSocket 与 HTTP 轮询必须支持平滑切换
