## 1. Python Standards (Backend + Agent)

### Core Principles

```python
# ✅ Good code
def get_contract_by_id(contract_id: UUID, tenant_id: UUID) -> Contract | None:
    """Retrieve a contract by its ID, scoped to the given tenant."""
    return Contract.query.filter_by(
        id=contract_id, tenant_id=tenant_id
    ).first()


# ❌ Bad code
def get_contract(id, tid):
    c = Contract.query.filter(Contract.id==id, Contract.tid==tid).first()
    return c
```

### Mandatory Rules

1. All functions must have type hints
2. All public functions must have docstrings
3. Line length ≤ 120 characters
4. Use `black` for auto-formatting
5. Use `ruff` for linting
6. Prohibit bare `except Exception` catches
7. LLM API calls must have timeout handling
8. All external API calls must be logged

### Agent Coding Standards

## 2. TypeScript/React Standards (Frontend)

### Mandatory Rules

1. Use ESLint + Prettier
2. Component files use PascalCase
3. Type definitions placed in `types/` directory
4. Prohibit `any` type (global config `noImplicitAny: true`)
5. API calls unified through `apiClient` wrapper
6. State management using Zustand (lightweight)

## 3. Git Commit Standards

```
Format: <type>(<scope>): <subject>

type:
  feat     - New feature
  fix      - Bug fix
  docs     - Documentation changes
  refactor - Code refactoring
  test     - Test changes
  chore    - Build/config changes

Examples:
  feat(analyzer): Add clause cross-validation logic
  fix(parser): Fix cross-page table parsing error
  docs(agent): Update Analyzer Agent Prompt template
```

## 4. Code Review Standards

| Check Item | Description |
|---|---|
| Agent Prompt Changes | Must have Legal Consultant Approval |
| Database Schema Changes | Must provide migration script |
| LLM API Calls | Must include error handling and retry logic |
| New Dependencies | Must explain selection rationale |
