## 二、直接竞品深度分析

### 2.1 幂律智能（PowerLaw AI）

| 维度 | 评估 |
|---|---|
| 成立时间 | 2017年 |
| 融资 | B轮（红杉、经纬） |
| 核心产品 | MeCheck 合同智能审查 |
| 技术路线 | NLP+规则引擎，2023年加入大模型 |
| 目标客户 | 大型律所和企业法务部 |
| 定价 | 面向大客户定制，非 SaaS 定价 |
| 优势 | 深耕法律 NLP 多年，判例库丰富 |
| 劣势 | 重心在大客户，中小企业覆盖弱；移动端体验差 |

**ContractGuard 应对策略：**
- 避其锋芒：不碰大律所市场，专注中小企业
- 差异化：SaaS 自助化 > 定制化项目实施
- 移动端：微信小程序 + Web 双端

### 2.2 华宇元典

| 维度 | 评估 |
|---|---|
| 成立时间 | 2014年（华宇软件子公司） |
| 核心产品 | 元典睿核（法律大数据平台） |
| 技术路线 | 法律知识图谱 + NLP |
| 目标客户 | 法院、检察院、司法行政机关 |
| 优势 | 司法场景独占，数据资源丰富 |
| 劣势 | 几乎不做商业合同市场；产品 TO-G 基因 |

**ContractGuard 应对策略：** 赛道不同，竞争关系弱。

### 2.3 法大大（合同审查模块）

| 维度 | 评估 |
|---|---|
| 成立时间 | 2014年 |
| 融资 | C+轮（腾讯、老虎基金） |
| 核心优势 | 电子签章市场份额第一 |
| AI能力 | 2023年引入大模型，但定位偏辅助 |
| 审查方式 | 关键词+模板匹配，非深度语义理解 |
| 优势 | 客户基数大，签章+审查一体化 |
| 劣势 | 审查功能浅，重签章轻审查；法务专业度不足 |

**ContractGuard 应对策略：**
- 定位差异化："他们是签章公司顺便做审查，我们是专门做审查的"
- 深度碾压：LLM 多步推理 > 关键词模板匹配

### 2.4 LawGeex（海外）

| 维度 | 评估 |
|---|---|
| 核心功能 | 英文合同自动审查 |
| 技术 | NLP + 预训练模型 |
| 市场 | 欧美法律市场 |
| 优劣势 | 不覆盖中文合同和中国法律体系 |

**ContractGuard 应对策略：** 无直接竞争关系，可参考其产品设计。

## 四、SWOT 分析

### Strengths（优势）
- 技术深度：多 Agent 协作 + 深度 RAG + Annotation 桥，非简单 Prompt 包装
- 场景聚焦：不做大而全，只做合同审查一件事
- 产品闭环：审查 → 内置编辑 → 一键修改 → 增量再审 → 打印，全流程系统内完成
- 成本优势：LLM API 成本远低于人工审查，按次 ¥19 无决策门槛
- 快速迭代：SaaS 模式，周级发布

### Weaknesses（劣势）
- 品牌为零：新品牌无信任积累
- 法律合规风险：需持续强调"辅助工具"定位
- 知识库建设：需要法务人员持续维护知识库

### Opportunities（机会）
- 90% 中小企业无自有法务，合同风险全靠运气
- 国内合同智能审查市场几乎空白
- 按次计费消灭了低频用户的订阅抵触心理
- 内置编辑器 + 打印避免了 Word/WPS 格式兼容问题

### Threats（威胁）
- 大厂入局（法大大已有签章用户基础）
- 通用 AI（ChatGPT/Kimi）持续进化
- 资金有限：初期依赖自有资金

### Opportunities（机会）
- 市场教育期：AI 概念火热，用户接受度高
- 政策利好："数字中国"政策推动企业数字化
- 中小企业蓝海：竞争对手都在抢大客户
- 出海机会：中文合同审查出海到东南亚华人企业

### Threats（威胁）
- 大厂入场：阿里云/腾讯云可能做法律 AI
- LLM 幻觉：一次严重错误可能毁掉口碑
- 用户习惯：中小企业习惯了找熟人律师
- 法规变化：AI 法律服务的监管政策不确定
